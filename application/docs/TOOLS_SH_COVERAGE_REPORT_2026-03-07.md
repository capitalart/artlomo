# tools.sh Comprehensive Coverage Report

**Date:** 2026-03-09
**Status:** ✅ Complete and verified

## Summary

The `/srv/artlomo/application/tools/app-stacks/files/tools.sh` orchestration script provides coverage for code, structure, system environment, runtime information, and curated AI handoff snapshots.

## Available Commands

```bash
./application/tools/app-stacks/files/tools.sh [command]
```

### Command Reference

| Command | Purpose | Output File Pattern |
| --- | --- | --- |
| `tree` | Folder structure snapshot | `application-folder-structure-{TIMESTAMP}.txt` |
| `stack` | Curated developer code stack | `application-code-stack-{TIMESTAMP}.md` |
| `gemini` | Curated single-file AI handoff stack | `application-gemini-code-stack-{TIMESTAMP}.md` |
| `backup` | Archive creation | `application-backup-{TIMESTAMP}.tar.gz` |
| `sysinfo` | System inventory report | `application-system-inventory-{TIMESTAMP}.md` |
| `all` | Run all four commands | (all of the above) |

Output directory: `/srv/artlomo/application/tools/app-stacks/stacks/`

## Output File Naming

All output files use a consistent timestamp format:

```text
{PREFIX}-{TYPE}-{DAY}-{DATE}-{MONTH}-{YEAR}-{TIME}-{AMPM}.{EXT}
```

Example:

```text
application-system-inventory-SAT-07-MARCH-2026-12-56-PM.md
```

This ensures:

- No filename collisions

- Clear chronological ordering

- Easy identification of report type

- Human-readable timestamps
